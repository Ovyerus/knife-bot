General
help
commands
[command]
Displays a list of all the currently available commands for the bot.
If command is specified, then the bot will try to display specific help for that command, if it exists.
Basic usage (list of all commands)
help
Recieving help for a specific command
help ping
info
stats
Shows general information about the bot's state, including servers, memory usage, and uptime.
invite
Sends a link to allow you to add the bot your own server.
linkcheck
checklink, check
<link>
Gets all the redirects (if any) of a given link.
Can be used to see if a link someone posted is suspicious or not.
ping
Shows the latency (in milliseconds) of the bot's connection to Discord.
This can be used as a simple command to check if the bot is online and/or responding.
support
Sends an invite to the official support server.
whois
lookup, user, inspect, member
<user>
Searches for a given user in the current server.
user can either be part of a user's name, their full name, their ID, or an @mention.
If you search using the user's name (or part of it), and there are multiple people with that in their name, a popup will appear, requiring you to choose the user to select.
Search by a part of a name
whois exam
Search using a full name
whois example
Search using an ID
whois 1234567890
Search using a mention
whois @example#1234
Moderation
halfway
softban
<user> [for <reason>]
Ban Members
Ban Members
Softbans a given user.
Softbanning bans a user and then quickly unbans them, and is an effective way to delete all the recent messages of a user, while removing them from the server.
Softbanning a user
halfway @example#1234
Softbanning a user with a reason
halfway @example#1234 for spam
mute
<user> [for <reason>]
Gives a user a muted role, preventing them from speaking.
This relies on roles set with the roles command. If there are no muted roles, this command will not work.
If there are multiple muted roles, you will be asked to select one.
Muting a user
mute @example#1234
Muting a user with a reason
mute @example#1234 for annoying
purge
<type> [options] [1-100]
Removes a bunch of messages, with optional filters and amount.
Muting a user
mute @example#1234
Muting a user with a reason
mute @example#1234 for annoying
roleban
<user> [for <reason>]
Rolebans a user, restricting them to a specific channel, allowing you to talk to them.
This relies on a role set with the roles command. If there is no roleban role, this command will not work.
Rolebanning a user
roleban @example#1234
Rolebanning a user with a reason
roleban @example#1234 for idiot

Settings
settings
Provides basic descriptions of the various commands for settings.
This used to be used for managing all the settings for the bot, however that was deemed to be too annoying and messy, so they all got split up into their own commands.
channel
[channel | disable]
Sets or removes the channel used for logging various events from the bot, including auto-mod filters being triggered, users being punished, etc.
Setting a logging channel
channel #mod-log
Removing the logging channel
channel disable
diacritics
[enable | disable | trigger <amount> | kick <amount> | ban <amount>]
Manages settings for the anti-diacritics filter.
enable
Enables the anti-diacritics filter.
disable
Disables the anti-diacritics filter.
trigger
<amount>
Sets the amount of diacritics in a single message that it takes to trigger the anti-diacritics filter.
kick
<amount>
Sets the amount of warnings to be issued to a user before they are kicked from the server, due to the anti-diacritics filter.
ban
<amount>
Sets the amount of warnings to be issued to a user, before they arre banned from the server, due to the anti-diacritics filter.
Cannot be lower or equal to the kick limit.
exceptions
[option [arguments]]
Manage exceptions for the filters.
channels
[add <channel> | remove <channel>
Manages the per-channel filter exceptions.
